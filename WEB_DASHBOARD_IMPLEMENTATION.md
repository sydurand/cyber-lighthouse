## Architecture

### Backend Structure

```
FastAPI Application (server.py)
├── Middleware
│   └── CORS (for local/LAN access)
├── Routes (from api/routes.py)
│   ├── GET /api/alerts
│   ├── GET /api/reports
│   ├── GET /api/stats
│   ├── GET /api/articles
│   ├── GET /api/system
│   └── GET /health
├── Static Files (./static)
│   ├── index.html
│   ├── css/style.css
│   └── js/
│       ├── api.js
│       └── app.js
└── Event Handlers
    ├── startup
    └── shutdown
```

### Frontend Architecture

```
Vue 3 Application (app.js)
├── State (ref)
│   ├── currentTab, isLoading
│   ├── alerts, reports, statistics, systemStatus
│   ├── allArticles, searchQuery, filterSource
│   └── alertsOffset
├── Computed Properties
│   ├── uniqueSources
│   └── filteredArticles
├── Methods
│   ├── refreshData() - Fetch all data
│   ├── updateCharts() - Render Chart.js
│   ├── autoRefresh() - Start polling
│   └── Watchers
├── Lifecycle
│   ├── onMounted() - Initial load
│   └── watch() - Data changes
└── HTML (index.html)
    ├── Navbar (tab navigation)
    ├── Sidebar (system status + quick stats)
    └── Main Content (4 panels)
        ├── Alerts
        ├── Reports
        ├── Statistics (with 2 charts)
        └── History (with search)
```

### Data Flow

```
1. User opens http://localhost:8000/
2. Frontend loads:
   - index.html
   - Bootstrap, Vue.js, Chart.js (CDN)
   - api.js (API client)
   - app.js (Vue application)

3. Vue app mounts and calls refreshData()
4. refreshData() makes parallel API calls:
   - /api/alerts
   - /api/reports
   - /api/stats
   - /api/system
   - /api/articles

5. API responses populate reactive state
6. Templates re-render with data
7. Charts are initialized with data

8. Auto-refresh (every 30 seconds):
   - Re-fetches all data
   - Updates charts
   - No page reload needed
```

## API Endpoints

### 1. GET /alerts
- Returns: Latest articles with analysis
- Parameters: limit (1-100), offset
- Response: AlertsListResponse

### 2. GET /reports
- Returns: Daily synthesis reports
- Parameters: limit (1-50), days (1-30)
- Response: ReportsListResponse

### 3. GET /stats
- Returns: System statistics
- Parameters: None
- Response: StatisticsResponse

### 4. GET /articles
- Returns: Searchable articles
- Parameters: search, source, date_from, date_to, limit, offset
- Response: ArticlesListResponse

### 5. GET /system
- Returns: System health and metrics
- Parameters: None
- Response: SystemStatusResponse

### 6. GET /health
- Returns: Server status
- Parameters: None
- Response: {"status": "healthy", "service": "..."}
